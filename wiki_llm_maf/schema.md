# LLM Wiki — Page Format Reference

> This file defines the format conventions for wiki pages.
> Agents read this to know how pages should be structured.
> It does NOT contain workflow logic or orchestration instructions.

<!-- SECTION:core -->

## Language Rule

All wiki content MUST be written in **English**, regardless of the language of the source
material. If a source document or Q&A answer is in another language, translate the content
to English when writing wiki pages.

---

## Directory Layout

The wiki root directory is configurable via the `WIKI_ROOT_DIR` environment variable (in `.env`). If not set, it defaults to the project directory. All paths below are relative to this root.

```
<WIKI_ROOT_DIR>/
  wiki/
    index.md        — Catalog of all pages (title + one-liner each)
    log.md          — Append-only timeline of operations
    sources/        — One page per ingested source
    entities/       — Pages for named entities (people, tools, companies, projects)
    concepts/       — Pages for abstract concepts
    synthesis/      — Pages generated from approved query answers
  raw/              — Source documents to ingest
  questions_approved/  — Approved Q&A to ingest as synthesis
  questions_pending/   — Q&A generated by the querier (awaiting approval)
```

---

## Frontmatter (all pages)

Every page MUST have YAML frontmatter:

```yaml
---
title: "Page Title"
type: "source|entity|concept|synthesis"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
sources: ["source-slug-1", "source-slug-2"]
---
```

- `updated` is refreshed on every modification.
- `sources` lists the source slugs that contributed to this page.

---

## Wikilink Format

Always use: `[[category/slug]]`

Examples:
- `[[sources/karpathy-llm-wiki]]`
- `[[entities/openai]]`
- `[[concepts/rag]]`

## Source Page (`wiki/sources/<slug>.md`)

```markdown
---
title: "<Source Title>"
type: "source"
source_file: "<filename in raw/>"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
entities: ["entity-slug-1", "entity-slug-2"]
concepts: ["concept-slug-1", "concept-slug-2"]
---

## Summary
<2-3 paragraph summary of the source>

## Key Takeaways
- Takeaway 1
- Takeaway 2

## Entities Mentioned
- [[entities/<slug>]] — brief role in this source

## Concepts Covered
- [[concepts/<slug>]] — how the source relates to this concept
```

---

## Entity Page (`wiki/entities/<slug>.md`)

```markdown
---
title: "<Entity Name>"
type: "entity"
entity_type: "<person|tool|company|project|other>"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
sources: ["source-slug-1", "source-slug-2"]
---

## Overview
<What this entity is, 2-3 sentences>

## From [[sources/<slug-a>]]
<What this source says about the entity — full detail, not summarized>

## From [[sources/<slug-b>]]
<What this other source says>

## Connections
- [[concepts/<slug>]] — relationship description
- [[entities/<slug>]] — relationship description
```

## Index Format (`wiki/index.md`)

The index is a content-oriented catalog. Each entry has a wikilink and a one-line summary
extracted from the page content, so the querier can locate relevant pages without opening them.

```markdown
# Wiki Index

## Sources
- [[sources/<slug>]] — one-line summary of the source document

## Entities
- [[entities/<slug>]] — one-line description of the entity

## Concepts
- [[concepts/<slug>]] — one-line definition of the concept

## Synthesis
- [[synthesis/<slug>]] — one-line summary of the synthesis
```

---

## Log Format (`wiki/log.md`)

Each entry:
```markdown
## [YYYY-MM-DD HH:MM] <operation> | <details>
<brief description of what was done>
Pages touched: <list>
```

<!-- /SECTION:index-log -->

## Contradiction Format

When a contradiction is detected, insert in the relevant page:

```markdown
> ⚠️ **Contradiction** (detected: YYYY-MM-DD)
> - Previous claim: "<claim>" ([[sources/<slug>]])
> - New claim: "<claim>" ([[sources/<slug>]])
> - Note: <resolution hint>
```
